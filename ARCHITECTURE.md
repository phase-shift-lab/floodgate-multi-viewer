# Architecture

更新日: 2026-07-17

## 全体構成

```text
GitHub Pages (React)
  ├─ src/domain: CSA/HTML/レート解析、選定、再生
  ├─ src/data: fetch、Abort、retry/backoff、fixture、保存
  └─ UI: 複数盤、ライブ追従、過去局検索・再生
           │ VITE_API_BASE
           ▼
Cloudflare Worker
  ├─ 固定ルートだけを許可
  ├─ CORS / 8秒timeout / 1回retry / edge cache
  └─ stale-if-error
           │
           ▼
wdoor.c.u-tokyo.ac.jp のFloodgate公開パス
```

ブラウザから上流への直接fetchはCORS実測で利用不可だったため、Workerをデータ境界にしています。WorkerはURL入力を受け取らず、`/api/today`、`/api/day?date=YYYY-MM-DD`、`/api/csa?file=検証済みCSA名`、`/api/ratings?date=YYYY-MM-DD`だけを固定ホストの既知パスへ変換します。

## データ境界

`src/domain` はDOM・React・ネットワークに依存しません。HTML一覧、レート表、CSAを正規化した `GameSummary` / `ParsedGame` に変換し、UIはその結果だけを扱います。不完全CSAは解析済み手数まで表示し、壊れた入力でアプリ全体を停止させません。

保存fixtureは2026-07-17の公開HTML、レート表、CSAから必要部分を縮小したものです。CSAは実対局の冒頭局面と終局マーカーを保持した再生・解析用fixtureで、完全棋譜の転載ではありません。ネットワーク障害時は取得層が要求された対局名・レート有無へfixtureを適合させ、画面に接続種別と取得時刻を明示します。HTTP 200でもCSAヘッダーを満たさないレスポンスは無効として扱います。

## 注目局の選定

同じ対局グループ内で次の順に採点します。

1. 両者レートの低い方が高い
2. 平均レートが高い
3. 開始時刻が新しい

ライブ局を優先し、選択済みAIには軽い重複ペナルティを加えます。不足枠は直近終了局で補完します。レートを解決できない場合は推測せず「不明」と表示します。

## ライブ更新

- 一覧: 表示中30秒、非表示120秒
- ライブCSA: 表示中3秒、非表示20秒
- 通信: `AbortController`、8秒timeout、指数バックオフ
- 過去局面を閲覧中: 新着を通知し、明示操作まで最新へ戻さない
- 終局後: 盤を保持し、自動切替設定が有効な場合だけ次局へ移る

Worker側でも一覧15秒、CSA 3秒、レート6時間のfresh判定を共有し、閲覧者数に比例して上流アクセスが増えにくい構成です。edgeには期限切れコピーを24時間保持し、上流失敗時は `X-FGMV-Stale: 1` として返します。ブラウザへはCSA `no-store`、一覧5秒、レート1時間の別ポリシーを返すため、ライブ更新をブラウザキャッシュが止めません。

## 状態とURL

表示局数、再生速度、盤面方向、テーマ、自動切替、お気に入り、最近見た対局をバージョン付きlocalStorageキーで保存します。共有URLには対局ID、手数、表示形式、盤面方向を含め、読み込み時に安全な範囲へ正規化します。

## 安全性とアクセシビリティ

- 秘密情報、広告、外部解析SDKを含めない
- Workerは固定ホスト・固定パス・入力検証・許可Originを使用
- Reactエラー境界、読込/空/障害状態、フォーカス表示を用意
- ボタンに判別可能な名前を付け、棋譜再生をキーボード操作可能にする
- `prefers-reduced-motion` 時は不要な動きを抑制する

## テスト方針

Vitestで盤面遷移を含むCSA解析、レート解析、選定、再生タイマー、保存復元、Workerの拒否規則を検証します。Playwrightでは実際のChromiumでデスクトップ4局、モバイル1局、切替、再生、手動移動、反転、復元、API障害、GitHub Pagesサブパスを検証します。
